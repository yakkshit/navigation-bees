BUMBLEBEE PROJECT - TIER Protocol 4.0
======================================

Author: Funda Yildiz Aydin
Date: 2026

PROJECT DESCRIPTION:
Automated tracking and analysis of single bumblebee movement from lab video recordings.
Videos are processed using TRex (background subtraction) to extract per-frame position data.
Tracking quality is then evaluated and results are analysed in a Jupyter notebook.

FOLDER STRUCTURE:
-----------------
BumblebeeProject/
├── README.txt                              → This file
│
├── Data/
│   ├── first.settings                      → TRex configuration file (tracking parameters,
│   │                                         camera calibration, detection settings)
│   └── InputData/
│       ├── Metadata/
│       │   └── experiment_metadata.txt     → Experiment details (date, arena, camera, bee info)
│       └── Videos/                         → Raw input video files (.mp4 / .mov)
│                                          
│
├── Output/
│   ├── trex_outputs/bee_results             → Per-video tracking folders generated by TRex
│   │                                         (each contains frame-level CSV files + trex.log)
│   ├── Results/
│   │   └── tracking_quality_summary.csv     → Per-video detection rate summary
│   └── Figures/                           
│
└── Scripts/
    ├── run_trex_batch.sh                   → Runs TRex on all videos in InputData/Videos/
    │                                         using background subtraction; outputs to trex_outputs/
    ├── check_tracking_quality.py           → Scans trex_outputs/, computes per-video detection rate,
    │                                         saves summary to Output/Results/tracking_quality_summary.csv
    ├── bee_analysis.ipynb                  → Jupyter notebook for analysing tracking results
    │                                         and generating figures
    └── MasterScript.py                     → Runs the full pipeline in order (start here)

DEPENDENCIES:
- TRex (conda environment: tracking)
- Python 3, pandas, jupyter

RAW DATA:
Large video files are available on computer

HOW TO RUN:
Step 1 — Track bees in all videos:
  bash Scripts/run_trex_batch.sh

Step 2 — Check tracking quality:
  python Scripts/check_tracking_quality.py

Step 3 — Analyse results:
  jupyter notebook Scripts/bee_analysis.ipynb



### New one### 

```
  # Full reprocess (recommended now that settings are improved):
  FORCE=1 bash Scripts/new_batch.sh

  # Reprocess Python steps only (no TRex re-run):
  bash Scripts/new_batch.sh

  # Skip already-done videos (default):
  bash Scripts/new_batch.sh

  # Keep full 1920×1080 resolution in output:
  FULL_RES=1 bash Scripts/new_batch.sh

```
